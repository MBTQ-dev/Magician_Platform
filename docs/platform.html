Platform Overview

MBTQ Platform provides the controller/platform layer for accessible applications. It consists of three core pillars that handle authentication, synchronization, and trust scoring.

Three Pillars

ğŸ”

DeafAUTH

/auth

Authentication & Identity Management
DeafAUTH provides Supabase-powered authentication with OAuth support, specifically designed for deaf-first accessibility. Handles user identity, session management, and secure authentication flows.

Key Features:
  • OAuth 2.0 authentication
  • JWT token management
  • User profile management
  • Event tracking and audit logs
âš¡

PinkSync

/sync

Real-time Communication & Accessibility
PinkSync manages real-time data synchronization and accessibility optimizations. Ensures seamless communication between services and provides real-time updates with accessibility-first design principles.

Key Features:
  • Real-time data synchronization
  • WebSocket/Socket.io integration
  • Accessibility-optimized messaging
  • Visual notification system
â­

Fibonrose

/trust

Reputation & Trust Scoring
Fibonrose implements a Fibonacci-based reputation and trust scoring system. Tracks user contributions, community engagement, and builds trust metrics across the platform ecosystem.

Key Features:
  • Fibonacci-based scoring algorithm
  • Badge issuance and tracking
  • Contribution recording
  • Community trust metrics

Integration Entry Points

For apps like vr4deaf.org

Integrate MBTQ Platform services into your application by connecting to the Supabase DeafAUTH project.

DeafAUTH Endpoints

GET /deafauth/profile

Retrieve user profile and authentication status

POST /deafauth/profile

Update user profile information

GET /deafauth/events

Access authentication event logs and audit trail

Supabase Configuration

Point your application to the DeafAUTH Supabase project:

# .env file - NEVER commit these values to version control
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_KEY=your-service-key
